选题的主要内容

任务概述

使用Cocos完成游戏客户端开发，使用Node，Go，MongoDB完成服务器端开发。使用Nginx和阿里云完成生产环境的部署。

各个功能模块的描述(上)

  • 使用Cocos完成玩家手柄的开发，包括手柄摇杆和炮弹发射按钮。
  • 使用Cocos完成玩家装备系统的开发，实现装备的升级，装备设计后，玩家的坦克能力变强，外形变化。
  • 使用 JavaScript 完成玩家手柄客户端的扫码功能，扫码后进入战斗场景。

各个功能模块的描述(中)

  • 使用Cocos完成金币系统的开发，玩家能够充值后，改变自己账号内的金币余额。
  • 使用Cocos完成战斗场景的开发，这是最复杂的一个模块，要处理各种碰撞检测，玩家的链接，玩家的断开，手柄控制信息的响应。
  • 使用Node开发游戏的WebSocket服务。
  • 使用Go开发游戏的HTTP服务。

各个功能模块的描述(下)

  • 使用MongoDB作为服务器端的数据库。
  • 使用Nginx作为服务器端的反向代理。
  • 阿里云的虚拟云主机作为公网可访问的生产环境的服务器。

预期结果

整个游戏被部署在阿里云上，拥有公网IP，并配置的域名解析，玩家可以在任何时间，任何环境打开PC，扫描PC上的二维码，进入游戏进行对战和装备的升级。

可行性分析(上)

Cocos是一个很著名的游戏开发引擎，在游戏客户端，手柄控制器，战斗大屏上的开发不是最难。

本游戏的数据库设计很简单，所以用MongoDB可以很方便的实现基本的数据库读写功能。

可行性分析(下)

因为自己是前端工程师，对JavaScript语言十分熟悉，对于使用Node完成WebSocket服务器（仅转发消息）是有把握的。

Go语言很简单，使用它开发一个HTTP的服务器也不在话下。

重点与难点分析(上)

多房间系统，因为网络断开，需要完成断线重连的功能，要实现这个功能Node开发的WebSocket服务器仅仅转发玩家手柄的控制信息就不够了。

重点与难点分析(下)

对接真实的支付。玩家是真的能使用支付宝，微信支付进行充值。这需要一定时间去阅读支付宝，微信支付的开发者文档。
